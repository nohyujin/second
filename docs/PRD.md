# 프로젝트 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
### 1.1 프로젝트명
**HRD-Analyzer** (로컬 기반 HRD 만족도 원클릭 분석기)

### 1.2 배경
- **문제점**: 데이터 보안 문제로 인해 클라우드 사용이 제한적인 환경에서 HRD 교육 만족도 데이터를 효율적으로 분석할 필요성 대두.
- **해결책**: 외부 서버 전송 없이 사용자 PC(Localhost)에서 안전하게 실행되는 Python 기반의 데이터 분석 대시보드.
- **핵심 가치**: 보안성(Localhost), 자동화(원클릭 분석), 데이터 기반 의사결정.

### 1.3 목표
- 교육 만족도 설문 결과를 자동으로 분석해주는 대시보드 제공.
- 외부 서버로 데이터를 전송하지 않고 100% 로컬 환경에서 실행하여 데이터 보안 유지.
- 주요 지표(KPI) 및 상관관계 시각화를 통해 HRD 인사이트 도출 지원.

---

## 2. 제품 상세 사양 (Spec)

### 2.1 기능적 요구사항 (Functional Requirements)

#### F1. 데이터 입력
- **엑셀 업로드**: 사용자가 설문 결과가 담긴 `.xlsx` 또는 `.xls` 파일을 업로드할 수 있어야 함.
- **표준 컬럼**: '만족도', '강사평가', '이해도', '실무적용', '추천의향(NPS)' 등의 컬럼을 처리.

#### F2. 대시보드 및 KPI
- **핵심 지표 표시**:
    - 평균 만족도 (5.0 만점)
    - 평균 강사 평가 (5.0 만점)
    - NPS (순수 추천 지수, 추천의향 점수 기반)
- **데이터 미리보기**: 업로드된 원본 데이터를 접기/펼치기 형태로 확인 가능.

#### F3. 데이터 시각화
- **분포 분석**: 직급별 만족도 분포 등을 파악하기 위한 박스 플롯(Box Plot).
- **상관관계 분석**: '교육 이해도'와 '실무 적용도' 간의 관계를 보여주는 산점도(Scatter Plot).
- **워드 클라우드** (예정): 주관식 의견 텍스트가 있을 경우 시각화.

### 2.2 비기능적 요구사항 (Non-Functional Requirements)
- **보안**: 모든 데이터 처리는 로컬(`localhost`)에서만 이루어져야 하며, 외부 서버로 전송되지 않아야 함.
- **실행 환경**: Python이 설치된 일반 업무용 PC에서 실행 가능해야 함.
- **사용성**: 코딩 지식이 없는 사용자도 쉽게 사용할 수 있는 웹 기반 인터페이스(Streamlit) 제공.
- **유지보수**: 추가 비용 없이 Python 스크립트로만 동작하며 유지보수가 용이해야 함.

## 3. 기술 스택 (Tech Stack)
- **언어**: Python
- **UI 프레임워크**: Streamlit (빠르고 직관적인 웹 대시보드 구현)
- **데이터 처리**: Pandas (엑셀 데이터 가공 및 분석)
- **시각화**: Plotly Express (상호작용 가능한 차트 구현)
- **의존성 라이브러리**: `streamlit`, `pandas`, `plotly`, `openpyxl`

---

## 4. 사용자 흐름 (User Flow)
1. **실행**: 사용자가 터미널에서 `streamlit run app.py` 명령어를 입력하여 프로그램 실행.
2. **접속**: 브라우저가 자동으로 열리며 로컬 대시보드(`localhost`)에 접속.
3. **업로드**: 분석할 교육 만족도 엑셀 파일을 드래그 앤 드롭으로 업로드.
4. **분석**: 시스템이 자동으로 데이터를 분석하여 KPI 및 차트 생성.
5. **인사이트 확인**: 사용자가 시각화된 보고서를 통해 인사이트 확인.

---

## 5. 개발 로드맵 (MVP)
1. **환경 설정**: Python 및 필요 라이브러리 설치.
2. **기능 구현**: Streamlit과 Pandas를 활용한 `app.py` 핵심 기능 개발.
3. **테스트**: 샘플 엑셀 데이터를 활용한 기능 검증.
4. **문서화**: 사용 가이드(실행 방법) 작성.
